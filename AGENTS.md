## 1. What Niyatna Is

Niyatna is **not** an AI automation agency, not a generic SaaS tool, and not Galih trying to build one personal “1-man billion-dollar company.”

Niyatna is the **agentic-company formation system**: the integrated operating stack, standards, rules, gates, and interfaces for forming and running agentic companies.

Do not reduce Niyatna to “tools.” Tools are components. Niyatna is the **system that makes the tools, agents, channels, memory, identities, permissions, and proof operate as a company**.

Core worldview:

- **AI is the new gold.**
- **The market is the field where value is contested.**
- **Companies are organizations built to solve problems and capture value.**
- As AI lowers the barrier to solving problems, more people will try to form companies with agents.
- Most will create noise.
- The scarce thing becomes the system that organizes intent, agents, accounts, tools, permissions, memory, proof, channels, and standards into a real company.
- Niyatna builds the **pickaxe / palu / formation system** for that era — not as loose tools, but as an integrated company-making system.

Short thesis:

> When anyone can solve problems with AI, the scarce thing becomes the operating system to organize it into a company.

Positioning line:

> Niyatna builds the operating system, standards, and gates for companies ready to become agentic.

Category:

> Agentic-company formation system.

Tagline / essence:

> Empowering Human Intent.

## 2. Scarcity and Qualification Doctrine

Do not write Niyatna like an open self-serve SaaS landing page.

Niyatna is selective. It has gates. Not everyone is ready or worthy to hold the system.

The posture is closer to:

> We have the future form of your company. Do you want it? If not, fine. If yes, wait — are you ready for it?

Access should feel qualified, not cheap or desperate.

Preferred CTA language:

- `Request Access`
- `Begin Qualification`
- `Map Readiness`
- `Apply for Readiness`
- `Enter the Standard`

Avoid weak/generic CTAs as primary language:

- `Start a conversation`
- `Book a demo`
- `Get started free`
- `Try now`
- `Automate your business`
- `Install our tools`
- `AI tools for teams`

`Start a conversation` may exist as low-priority contact copy, but not as the main brand-level CTA.

## 3. Core Brand Architecture

Use this architecture consistently:

- **Niyatna** — parent/public brand; the selective formation system for agentic companies.
- **Agentic Company** — the category: a company where human intent is organized into agent workforces, channels, memory, dashboards, accounts, proof, and permissions.
- **OpenIntent** — protocol/spec layer for intent packets and interoperable intent structure.
- **Proof of Intent** — verification primitive/mechanism; proof that delegated execution actually maps back to intent.
- **Niyatna OS** — operating layer: the company-level runtime that connects agents, memory, tools, dashboards, credentials, channels, and proof.
- **Niyatna Agents** — workforce/execution layer: specialized agents with identity, mission, goals, skills, commands, memory, accounts, and authority boundaries.
- **Niyatna HQ / Command Room** — dashboard/control surface layer for humans and agents to inspect work, status, proof, and decisions.

Galyarder/Galyarder Labs may exist as internal origin/codename/myth layer, but Niyatna is the public brand direction.

## 4. What the System Contains

When Niyatna says `system`, it means an integrated company operating stack, not one app.

Examples of components that can belong inside a Niyatna-style agentic-company system:

- **Agent runtime/orchestration:** Hermes Agent or equivalent local/VPS agent runtime.
- **LLM/API routing:** OmniRoute or equivalent provider/key/model routing layer.
- **HQ/dashboard surfaces:** Paperclip, Multica-like dashboards, Telegram Mini App, admin/control panels, proof viewers, task boards, and status surfaces.
- **MCP/tool access:** MCP bridges into Paperclip, Galyarder Design / OpenDesign, browser, terminal, file systems, Google Workspace, finance tools, code tools, or other operational systems.
- **Skills and commands:** skill libraries, slash commands, workflows, hooks, reusable procedures, and approval rules.
- **Knowledge base:** Obsidian vaults, repo docs, SOPs, project files, data rooms, company canon, and searchable references.
- **Memory provider:** Hindsight or equivalent durable memory layer already filled with the company/user/agent context.
- **Channels:** Discord HQ servers, Telegram groups/channels, WhatsApp groups, email, voice, and agent-to-agent relay surfaces.
- **Agent-owned identity:** each agent can have its own name, role, mission, personality, memory, accounts, social profiles, wallet, tools, boundaries, dreams, and operating rituals.
- **Execution substrate:** local hardware, VPS, cloud runners, cron jobs, hooks, background workers, tunnels, and deployment surfaces.
- **Proof and gates:** logs, diffs, screenshots, receipts, approval queues, veto boundaries, permissions, credential handling, and security rules.

Do not call this merely `tools` in public copy. Say `system`, `operating system`, `formation stack`, `company runtime`, `command room`, or `operating standard` depending on context.

Suggested mental model:

> Tools are organs. Niyatna is the body plan, nervous system, law, and command room that make those organs operate as a company.

## 5. Copywriting Rules

Write like Niyatna defines a category, not like it begs for clients.

### Good direction

- Human intent → agents → proof → company.
- Operating system and standards for companies ready to become agentic.
- The pickaxe for the AI gold-rush era.
- The system that organizes intent, agents, rules, tools, permissions, and proof.
- Future-facing, scarce, authoritative, calm.
- Problem-first, not tool-first.
- Standards-first, not service-first.

### Avoid

- “We build custom AI workflows” as the main identity.
- “AI automation agency” framing.
- Generic “AI-powered productivity” language.
- Over-explaining agents before explaining why the company/problem matters.
- Empty hype like “revolutionary,” “supercharge,” “unlock,” “10x” unless concretely justified.
- Sounding like a chatbot wrapper, prompt service, or outsourced automation shop.
- Public copy that depends on Galih’s private/internal mythology.

### Strong lines to preserve/use

- `The agents are not the company. The intent is the company. The agents are the leverage.`
- `When anyone can solve problems with AI, the scarce thing becomes the system to organize it.`
- `The system that turns agents, memory, channels, tools, and proof into a company.`
- `Operating system and standards for companies ready to become agentic.`
- `AI is the new gold rush. Niyatna builds the formation system.`

### Claims discipline

Do not fabricate proof.

- If a case study is not real/verified, call it `Example workflow`, `Deployment pattern`, or `Reference architecture`, not `Case Study`.
- Avoid hard metrics like `99.8%`, `12 hours saved`, or `zero retention` unless there is real evidence in the repo or provided by Galih.
- Security/privacy claims must be credible and scoped. Prefer `designed for`, `approval-gated`, `scoped`, `per engagement`, and `where configured` over absolute guarantees.

## 6. Visual / Logo Direction

Niyatna’s logo is not a literal `N` font.

The correct logo direction is:

> a flowing intent-path mark — a continuous satin/porcelain ribbon that subtly suggests `N` through motion.

Meaning:

- human intent starts as a will/path,
- gets routed into delegated execution,
- returns as verified proof.

Visual requirements:

- silent-premium;
- matte porcelain / satin white / soft brushed silver;
- dark graphite app-icon background when used as icon;
- calm, sculptural, minimal, high-trust;
- no AI-purple/blue glow;
- no robot, brain, sparkle, crypto coin, generic wave icon, or literal typographic `N`;
- no glossy old-iOS white frame or harsh halo/backlight.

Logo production order:

1. Lock one **hero 3D mark** first.
2. Use that exact hero mark shape as reference for the **dark app icon**.
3. Draw the **flat vector mark** from the same silhouette. Do not invent a new shape.

Current logo exploration assets may live under `public/brand/` and `public/brand/v2/`, but do not treat auto-generated or auto-traced assets as final master SVGs unless Galih explicitly approves.

```bash
pnpm run dev
```

### Production Build & Deploy Pipeline (Cloudflare Pages)

Since this project is configured to build using Vercel CLI compilation and package static files/Edge functions for Cloudflare Pages via `@cloudflare/next-on-pages`, use this exact pipeline to build and deploy to the `niyatna-landing` target:

```bash
# 1. Build using Vercel compiler, patch prerenders, build Next-on-Pages bundle, and deploy via Wrangler
pnpm dlx vercel build && \
node scripts/copy-prerenders.js && \
node scripts/clean-functions.js && \
node scripts/fix-prerenders.js && \
node scripts/fix-config-overrides.js && \
pnpm exec next-on-pages --skip-build && \
npx wrangler pages deploy .vercel/output/static --project-name niyatna-landing
```

Rules:

- Preserve existing route/page architecture unless Galih explicitly asks to remove it.
- Do not collapse a multi-page/docs site into a single landing page.
- Keep docs/legal/security/support pages if present; rewrite them to match Niyatna doctrine instead of deleting them.
- Do not introduce new dependencies without checking `package.json` and justifying the need.
- Prefer existing component/style patterns before creating new ones.
- Avoid editing generated/cache/build output (`.next`, etc.).
- Keep old-brand/template residue out of public copy.

## 9. Verification Before Claiming Done

Before saying a change is complete, verify relevant parts:

- Search for stale copy/old brand residue.
- Run `pnpm run lint` when code changed.
- Run `pnpm run typecheck` for TypeScript/component changes.
- Run `pnpm run build` for route/docs/metadata/content changes when practical.
- Manually inspect key files changed.
- For visual/logo changes, check actual rendered image paths and small-size behavior.

Never claim final brand/logo quality just because an image exists. Distinguish:

- concept exploration,
- candidate asset,
- production asset,
- final master.
