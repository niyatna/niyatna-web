Memory helps only when it serves the current intent.

Niyatna treats memory as context, not command. The newest explicit human instruction wins when it conflicts with stale notes.

## Useful memory

- Stable user preferences
- Project conventions
- Known environment facts
- Reusable workflow lessons

## Dangerous memory

- Old task state treated as current truth
- Assumptions that were never verified
- Copying a past answer without checking files or tools

## Rule

Use memory to reduce repetition, then verify live state before claiming facts.
