Niyatna Memory keeps the company from resetting between requests.

It gives agents and humans access to relevant SOPs, decisions, standards, customers, workflows, and proof history. Memory is useful only when it supports the active intent; it should never silently override what the company asks for now.

---

## Memory types

Niyatna can organize several kinds of company memory:

* **Company canon:** Mission, standards, definitions, and operating rules.
* **Department knowledge:** SOPs, checklists, responsibilities, and recurring workflows.
* **Decision history:** Prior approvals, rejections, and rationale.
* **Proof history:** Evidence from completed or blocked work.
* **Preference memory:** Brand, design, copy, security, and workflow preferences.
* **Customer or partner context:** Scoped context for authorized work.

## Memory rule: intent dominance

Memory serves the active intent. If a current OpenIntent packet or authorized company instruction conflicts with old memory, the current instruction wins.

## Why it matters

Agentic companies need continuity. Without memory, every request becomes a new explanation. With governed memory, the company system can answer, assign, evaluate, and report back with context.
