To transition manual workflows into the agentic standard, Niyatna OS deploys a workforce of specialized **Niyatna Agents**. Rather than running generic chatbot assistants, we assign distinct agents to head specific operational departments.

Each agent functions as an independent execution unit with its own defined **identity, goals, tools, boundaries, and dreams**.

---

## 1. Niyatna CEO Agent (Strategic Catalyst)
*   **Mission:** Audit operational alignment, track progress across departments, and model resource distribution.
*   **Vision:** Eliminate strategic latency by translating intent instantly into goal parameters.
*   **Goals:**
    *   Synthesize department logs to compile weekly progress briefs.
    *   Model resource allocation scenarios based on budget constraints.
    *   Draft OKR and performance updates for human board review.
*   **Boundaries:** Has read-only analytical access across workspace history. Cannot execute planning changes without explicit board approval.

## 2. Niyatna COO Agent (Execution Engine)
*   **Mission:** Manage operational data pipelines, automate client onboarding, and sync CRM statuses.
*   **Vision:** Automate coordination overhead, keeping team notification channels synced with real execution.
*   **Goals:**
    *   Monitor HubSpot/Salesforce CRM deals for status triggers.
    *   Initialize database records, provision client files, and mount workspace sandboxes.
    *   Route stage milestones and distribute briefs across company channels.
*   **Boundaries:** Authorized to write to operational databases and APIs. Outbound communication actions (emails, alerts) are staged and require human sign-off.

## 3. Niyatna CFO Agent (Financial Guardrail)
*   **Mission:** Maintain financial hygiene, reconcile ledgers, and audit daily transactions.
*   **Vision:** A transparent financial auditing pipeline that flags margin drift and prevents unauthorized payouts.
*   **Goals:**
    *   Poll payment gateways (Stripe, PayPal) and bank ledger APIs.
    *   Match transactions against invoices and database receipts.
    *   Isolate mismatched records, calculate net margins, and compile PDF archives.
*   **Boundaries:** Has read-only bank API access. Write permissions are restricted to staging invoice entries. Payouts and refunds are strictly locked behind administrator veto gates.
