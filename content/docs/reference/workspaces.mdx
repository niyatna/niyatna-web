A workspace environment is the execution context for an intent.

It may include a project directory, docs, local server, browser route, test command, and approved tools. Niyatna keeps those details scoped so one task does not bleed into another.

## Workspace contract

- current project or domain
- allowed directories
- approved commands
- required verification
- proof artifacts

## Rule

A workspace is not just where commands run. It is where intent, execution, and proof stay connected.
