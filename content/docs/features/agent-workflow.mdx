A **Niyatna Agent lifecycle** begins when an intent is assigned and ends when the company receives proof and report-back.

Agents are useful because they can carry work forward, but they must remain bounded by role, context, permissions, and proof requirements.

## Lifecycle phases

1. **Intent intake:** The agent receives the goal, context, permissions, and acceptance criteria.
2. **Route selection:** Niyatna chooses the right model, tool, data source, or human handoff.
3. **Execution:** The agent performs the scoped work.
4. **Self-check:** The agent gathers evidence and checks the work against the acceptance criteria.
5. **Review gate:** Risky or external actions pause for approval.
6. **Report-back:** The result, proof, and memory update return to the company system.

## Bounded autonomy

Niyatna Agents can use powerful tools, but authority stays legible. The company should always know what the agent was asked to do, what it touched, and what proof it returned.
