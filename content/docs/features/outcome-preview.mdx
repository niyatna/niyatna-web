Outcome preview (the preview pane) shows the result where the context still exists.

For web UI, docs, dashboards, automations, and generated artifacts, the output should be inspectable next to the original intent packet.

<img src="/web_preview.webp" alt="Niyatna proof preview" width="2560" height="1600" />

## Preview evidence

- Routes opened
- Viewports tested
- Console errors checked
- Screenshots captured
- Interaction states verified

## Rule

A preview is not decoration. It is evidence.
