The **Outcome Preview** panel inside Niyatna HQ provides a live visual rendering of agent execution before commit.

For frontend UI, docs, reports, and generated static pages, the layout shows the result alongside the original OpenIntent requirements and proof constraints.

## Preview Parameters

The preview interface displays:
*   **Active Route Checks:** Confirming all generated endpoints return valid responses.
*   **Viewport Testing:** Showing page designs across mobile, tablet, and desktop sizes.
*   **Console Auditing:** Exposing terminal warnings, warnings, and compilation logs.
*   **Visual Artifacts:** Captured screenshot evidence proving visual correctness.
*   **Interaction Verification:** Verifying scroll effects and state changes.

## Rule: Visual Evidence

A preview is not decoration. It is visual proof. Niyatna HQ ensures that visual outputs are fully inspectable, eliminating guess work before deployment.
