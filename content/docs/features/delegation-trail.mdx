The delegation trail (change control panel) keeps agent work reviewable.

Niyatna treats every meaningful edit as part of a proof trail: what changed, why, which command or decision caused it, and how it satisfies the intent.

<img src="/source-control.webp" alt="Niyatna change control" width="2560" height="1600" />

## Review checklist

- Did the changed files match the intended scope?
- Did any unrelated component get touched?
- Are tests, builds, or screenshots attached?
- Are unresolved tradeoffs named clearly?

## Result

The human can approve with evidence instead of redoing the whole investigation.
