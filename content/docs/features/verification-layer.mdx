The verification layer (accessed via the proof editor) is the review surface for agent output.

It is where proposed changes, notes, diffs, references, and acceptance criteria can be inspected together instead of scattered across chat history.

<img src="/editor.webp" alt="/editor.webp" width="2560" height="1600" />

## What belongs here

- Drafted copy, specs, code, and docs
- Diffs and suggested changes
- Source references and evidence
- Open questions and tradeoffs
- Approval notes

## Principle

If a human cannot inspect it, it should not be treated as proof.
