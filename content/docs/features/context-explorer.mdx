The **Context Explorer** in Niyatna OS ensures that agents execute with complete business context, eliminating guesswork and prompt trial-and-error.

It compiles project directories, schema files, known guidelines, and historical knowledge, delivering a clean context window to the agent runtime.

---

## Memory & Context Layers

Niyatna OS handles two core context variables:

*   **Static Context:** Project structures, API specifications, and codebases.
*   **Local Memory:** Persistent history of past operational decisions, preferences, and verified patterns, ensuring the agent does not repeat historical errors.

## Rule: Intent Priority

Context and memory are secondary to intent. If an instruction in the OpenIntent packet conflicts with past memory, the newest human command wins. The Context Explorer filters memory dynamically, injecting only what is relevant to the active goal to prevent context drift.
