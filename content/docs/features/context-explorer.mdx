The context explorer keeps the agent from guessing.

It gives the work lane a structured set of files, sources, notes, and references the agent is allowed to use.

## Context types

- Project files and directories
- Product docs and previous decisions
- Screenshots and visual references
- External sources and URLs
- User constraints and taste notes

## Boundary

Context is not a dumping ground. Only include what helps the current intent. Too much irrelevant context creates drift.
